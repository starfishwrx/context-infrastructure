# X6. 抓大放小系统分析

## 1. 核心公理

面对高价值、陌生且复杂的系统，先问什么机制、业务结构或运行约束生成了表面现象，再找最大决策点。在寻求答案前积累问题，与假设相悖的地方通常是最有价值的信号。

## 2. 深度推演

### 方法论步骤

1. **先找生成机制**：追问表面现象由什么结构、激励、约束或因果链产生。这里的目标是建立可检验模型，而不是用一句“本质就是”提前结束探索。

2. **再找最大决策点**：不要试图理解一切，先抓住真正改变结果的主干决策。找到它们比覆盖所有细节更重要。

3. **持续提问并聚合**：问题本身就是线索。把多个问题聚合起来，可以定位共同的认知缺口和根因候选。

4. **关注与假设相悖的地方**：最不直观的证据可能暴露模型缺口、隐藏约束或真正的改进机会。

### 与其他方法的关系

本方法与渐进式披露（从粗到细）和文档优先（理解优先，文档跟随）互补。本源文件中的 "Data vs Opinion 决策原则" 已由 T04（数据优于观点）覆盖。

## 3. 应用判定

适用场景：
- 面对陌生系统或新领域
- 处理遗留代码库或复杂架构
- 文档缺失或不完整的项目
- 需要快速建立系统全景认知

边界条件：
- 简单、低风险、可逆且结果容易验证的任务，直接执行并检查结果，无需先做完整机制分析
- 高价值、高不确定或错误代价较高的任务，先建立生成机制和关键假设，再行动
- 机制解释必须能接受反例和证据校准，避免把“寻找本质”变成过早简化

# 内部文档排版与自适应视觉组件规范

## 元数据

- **类型**: BestPractice
- **适用场景**: 面向用户、内部协作者或未来 AI Agent 的内部文档（Memo/RFC/周报）视觉化与排版。
- **创建日期**: 2026-07-06

## 核心设计哲学

内部视觉化的根本目标是：**用形式独立承载价值，减少读者读字的时间，加速决策与审计速度**。
Visual 是提升认知带宽的工具，不是装饰。

---

## 1. 视觉武器库 (Visual Components)

选择最合适的视觉武器，解决特定的认知问题。切忌为了视觉化而视觉化：

| 视觉武器 | 解决的认知问题 | 适用场景 |
| :--- | :--- | :--- |
| **短段落** | 默认形态，承载精细论证与逻辑推理。 | 几乎所有详细说明。 |
| **紧凑表格** | 多个项目之间稳定维度的对比（触发空间记忆）。 | 项目 $\ge 4$ 且具备一致的数据结构。 |
| **卡片网格** | 少数并列项的状态与 Takeaway（色块直观判定）。 | 项目数 $\le 4-6$ 且色块状态分布均匀。 |
| **chip / pill** | 标记单一项的类型或状态语义（平行扫读通道）。 | 同列类别分布 $\ge 2$ 种，颜色语义就地解析。 |
| **`<details>` 折叠** | 审计材料、大段日志、实现细节、历史废弃方案。 | “读者可能想验证，但不属于首屏决策”的内容。 |
| **Anchor 链接** | 建立“声明 $\rightarrow$ 证据”的就地跳转。 | 任何得出核心结论、需要源文件/代码佐证的句子。 |

## 2. 局部 HTML-in-Markdown 增强与自适应机制

在 Markdown 中可以使用 HTML/CSS 实现卡片、左右对照、状态条，但必须保证其在移动端及 dark/light 模式下双向自适应。

### 约束一：只做局部增强与显式闭合
*   严禁使用一个外层 `<div>` 包裹整篇文档，只对单个卡片或组件设置样式。
*   必须显式闭合每个 HTML 标签（如 `</div>`），严禁泄漏导致整篇文档排版和颜色漂移。

### 约束二：CSS 颜色必须使用「主题变量 + fallback」
为了防止在 Dark Mode 下白底黑字或深底深字，必须使用 `var(--主题变量, 回退值)`。
*   **常用变量与回退值对照**：
    *   正文颜色：`var(--fg, #1a1a1a)`
    *   背景颜色：`var(--bg, #ffffff)`
    *   次要文字：`var(--fg-muted, #666666)`
    *   卡片底部背景：`var(--card-bg, #f6f6f6)`
    *   边框：`var(--border, #e5e5e5)`
    *   语义状态色（绿/成立）：`--ok-border` (`#10b981`), `--ok-bg` (`#d1fae5`), `--ok-fg` (`#065f46`)
    *   语义状态色（红/否定）：`--bad-border` (`#ef4444`), `--bad-bg` (`#fee2e2`), `--bad-fg` (`#991b1b`)
*   **书写示例**：
    ```html
    <div style="border-left: 5px solid var(--ok-border, #10b981); background: var(--card-bg, #f6f6f6); color: var(--fg, #1a1a1a); padding: 10px;">
      <strong>业务结论</strong>: 方案 A 成立。
    </div>
    ```

### 约束三：语义类必须使用复合选择器
装饰性的行内元素（如 chip/badge/pill）在设置 CSS 类时，必须绑定双类（如 `.vx-chip.ok` 或 `.chip.ok`），而不能只使用裸单类（如 `.ok`）。这样可以保证特异性高，不被父容器的 `--fg` 覆盖。
*   *正确示范*：`class="vx-chip ok"`
*   *错误示范*：`class="ok"`

## 3. 常见视觉化反模式 (Anti-patterns)

在设计或输出内部文档前，请对照以下失败模式自检：

1.  **卡片当百科条目写**：在卡片里塞满 150 字以上的长文字。卡片只适合存放 Chip + 标题 + 一两句判决。长段落应该从卡片里拿出来，放在卡片下方的 Markdown 正文中。
2.  **硬编码浅色背景**：如 `background: white` 或 `background: #f8fafc`。这会在 Dark Mode 下形成一块晃眼的白斑，且白底上的白色文字彻底不可见。
3.  **信息墙**：一段内塞入 5 个以上的数据指标。请拆成简短的 Key-Value 列表或 2 列的小表格。
4.  **内联 SVG 图片**：不要在最终 Markdown 中夹带内联 SVG 代码，移动端渲染极其不稳定。需要图时，渲染成 PNG/JPG 引用。
